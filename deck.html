MeshCredit

When an agent moves money,
who vouches for it?

No portable agent identity
AI agents have no standardised credential a bank or protocol can verify. Every integrator reinvents a local check — or skips it.
Trust stored, not enforced
Today's solutions are rows in vendor databases, checked by APIs that sit above the money rails and can be bypassed by a buggy or malicious integrator.
Accountability is unlinked
When an agent misbehaves, there is no on-ledger trail to the organisation that deployed it. No traceable principal, no recourse, no kill-switch.
No data flywheel for trust
Agent behaviour never aggregates into a portable reputation. Each payment is a fresh cold-start — even for an agent with a clean two-year history.
Trust is the binding constraint on autonomous agent payments.

Under the passport

Six dimensions. Each has a named issuer
and independently checkable evidence.

D1 · Principal / KYB
Organisation accountability
Zefix (Swiss registry)
Demo: Novartis AG, UID CHE-103.867.266. Cached real public-registry fixture. Anti-impersonation: claim valid only if principal counter-signs agent key.
D2 · Human accountability
Flesh-and-blood link
World ID IDKit v4
Orb-verified human behind the deployment. Testable via simulator in demo.
D3 · Code & runtime provenance
Code-swap detection
SHA-256 (self-verifiable)
Recomputable hash of agent harness + skill. Anyone can recompute. A patched binary fails the check before the credential is re-issued.
D4 · Behavioral record
On-chain settlement history
Public XRPL ledger
Re-derivable from third-party on-chain data. No MeshCredit trust required — an auditor derives the same score independently.
D5 · Capability & scope
Signed spending mandate
Principal-signed
Effective ceiling = min(tier ceiling, principal's remaining delegation budget). Delegation chain is on-ledger.
D6 · Compliance / AML
Hard gate — OFAC SDN screen
US Treasury SDN list
Full 19,794-entry list. A match is AML-DENY — payment blocked at payment-assessment time. DeepSeek call #5 reasons about the AML hold. Hard gate: fires during every payment attempt, not only at cert issuance.

Business model · GTM · path to Mainnet (40% viability)

Capital-light trust bureau.
We issue the score; the bank keeps the decision.

Revenue stream 1

Relying-party verification

Banks, PSPs, DeFi venues pay per credential read at transaction time. Zero marginal cost per check; network value scales with agents onboarded.

Revenue stream 2

Onboarding + renewal fees

Operators pay to issue and renew agent passports. Kill-switch SLA (revoke once, denied everywhere) is an enterprise upsell.

Moat layer 1 — data

Behavioral consortium

Negative-event sharing across relying parties. The graph grows with usage — a private database cannot replicate public XRPL auditability.

Moat layer 2 — network

Reputation portability

The passport moves with the agent across rails and venues. Locking an agent into a single platform is not possible when the credential lives on a public ledger.
