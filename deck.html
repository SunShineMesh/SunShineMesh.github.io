The problem

When an agent moves money,
the bank can't tell if it's trustworthy.

So every cross-border transfer lands in the manual review queue. Agent autonomy dies at the compliance wall — and there's no portable, verifiable way to vouch for a machine.

Trust is the binding constraint on agent payments.
The insight

Don't store trust in a database.
Enforce it at consensus.

Everyone else

A layer up

Trust is a row in a vendor DB, checked by an API the relying party must trust, enforced in application code above the money. Skippable by any buggy or malicious integrator.

MeshCredit

In the ledger

The credential and the payment gate are the same primitive. The validators reject an uncertified payment. The rule cannot be skipped.

The product

The on-ledger trust passport.

An XLS-70 credential carrying the agent's tier, max single payment, content-attestation hashes and DID — readable by any bank, PSP or venue straight from the ledger, no API key.
